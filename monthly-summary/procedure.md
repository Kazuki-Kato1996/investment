# 月次取引・収支まとめ手順書

## 概要
毎月末に実施する、Googleスプレッドシート上での月次取引と収支の集計・新月シート作成作業。

## 対象スプレッドシート
- URL: https://docs.google.com/spreadsheets/d/12hycp-InFw3fGUkyMjdW9lvOWF8mOX6pWfJbEVV2RKk/edit
- スプレッドシートID: `12hycp-InFw3fGUkyMjdW9lvOWF8mOX6pWfJbEVV2RKk`

### 関係シート
| シート | gid | 役割 |
|---|---|---|
| 年間集計 | 1181632804 | 月次サマリー（行4=1月, 行5=2月, ..., 行8=5月, 行15=12月） |
| 2026.M（当月） | 各月で異なる | 当月の銘柄別ポジション・株価・損益管理 |
| 2026.M+1（翌月） | 新規作成 | 当月をコピーして翌月用に作成 |

## 実施タイミング
毎月末（翌月分のシートを作成する形）

## 作業手順（4月→5月の例。来月以降は月数字を読み替える）

### Step 1. 当月シート（2026.4）の数値を確定（フリーズ）

当月シートには株価・配当の動的フォーミュラが入っているため、月末値を固定値に置き換えて確定させる。

対象列（2026.4 シート）:
- **K列**: 現在株価（`STOCKPRICEJP`/`GOOGLEFINANCE`/`IMPORTXML` の数式）→ 値貼り付け
- **L列**: 株価関連の派生値 → 値貼り付け
- **R列**: 配当関連（`IMPORTHTML` の数式）→ 値貼り付け

操作:
1. 2026.4 シートで K列のデータ範囲（K10〜末尾）を選択 → コピー → 「値のみ貼り付け」（Cmd+Shift+V）
2. 同様に L列の該当範囲を値貼り付け
3. 同様に R列の該当範囲を値貼り付け

### Step 2. 翌月シート（2026.5）の作成

1. 2026.4 タブを右クリック → 「コピーを作成」
2. 新しいタブの名前を `2026.5` に変更

### Step 3. 翌月シート（2026.5）の前月株価・前月評価額をリセット

新月の月初基準を更新する（コピー元は4月初時点の値が残っているため）。

操作（2026.5 シート）:
1. **V列（前月株価）**: K列（4月末の確定価格）をコピー → V列に値貼り付け
2. **W列（前月評価額）** は `=V*G` の数式なので自動で再計算される
3. **U列（月初比%）** は `=(K-V)/V` の数式なので自動で 0% になる（月初時点で K=V のため）

※ V列/W列に保有株数や銘柄追加分があれば併せて調整

### Step 4. 年間集計の翌月行（5月＝行8）を入力

行8の各列に以下の数式を入力。基本パターンは「行7（4月）の数式をコピーして、参照先を `'2026.4'` から `'2026.5'` に置換」。

| セル | 数式 | 備考 |
|---|---|---|
| B8 | `5月` | ラベル |
| C8 | `='2026.5'!J144` | 取得原価 |
| D8 | `='2026.5'!L144` | 評価益 |
| E8 | `='2026.5'!K144` | 時価評価額 |
| F8 | `='2026.5'!J137+'2026.5'!J138` | 住信SBI |
| G8 | `2183569`（手入力の固定値） | SBI現金 ※当月の実残高に更新 |
| H8 | `='2026.5'!J146` | SBI米株 |
| I8 | `='2026.5'!O144` | 売却損益 |
| J8 | `=E8+G8+N8+H8+F8` | 資産合計 |
| K8 | `=D8/C8` | 評価率 |
| L8 | `=(J8-J7)/J7` | 増減率（前月比） |
| M8 | `=SUM($I$4:I8)/10000000` | 累計売却損益 |
| N8 | （空欄または入金額を記入） | 入金 |
| O8 | `=P8*$J$3/$P$3+1000000+2000000` | 日経平均換算 |
| P8 | 月末日経平均値（手入力） | 例: `59284` |
| Q8 | `=(P8-P7)/P7` | 日経増減率 |
| R8 | `=($P8-$P$3)/$P$3` | 日経年間 |
| S8 | `=T8*$J$3/$T$3` | TOPIX換算 |
| T8 | 月末TOPIX値（手入力） | 例: `3727.21` |
| U8 | `=(T8-T7)/T7` | TOPIX増減率 |
| V8 | `=(T8-$T$3)/$T$3` | TOPIX年間 |
| W8 | `=X8*$J$3/$X$3` | SP500換算 |
| X8 | 月末SP500値（手入力） | 例: `7117` |
| Y8 | `=(X8-X7)/X7` | SP500増減率 |
| Z8 | `=(X8-$X$3)/$X$3` | SP500年間 |
| AA8 | `5月` | ラベル |
| AB8 | `=AC8*$J$3/$AC$3` | SBI米国＄換算 |
| AC8 | `='2026.5'!K212` | SBI米国＄ |
| AD8 | `=AE8*$J$3/AE$3` | WealthNavi換算 |
| AE8 | 月末WealthNavi値（手入力） | 例: `12113` |
| AF8 | `=AG8*$J$3/AG$3` | folio換算 |
| AG8 | 月末folio値（手入力） | 例: `11599` |

#### 手入力が必要な値（ユーザー入力）
- **G8**: SBI現金残高（当月末時点）
- **N8**: 当月入金額（あれば）
- **P8**: 月末日経平均
- **T8**: 月末TOPIX
- **X8**: 月末S&P500
- **AE8**: 月末WealthNavi評価額
- **AG8**: 月末folio評価額

### Step 5. 年間集計の補助行（行47）を入力

行47には行8（当月）の値を分解して再表示する集計行がある。`=J8` などで月次値を引き、AB47以降の細目から自動計算される（数式は前月行をコピーすればOK）。

例:
- C47: `=J8`（資産合計を引く）
- D47: `=AB47+AC47`
- E47: `=AD47`
- ... 以下、行46の数式を行47にコピーして相対参照を効かせる

### Step 6. 年間集計のSUMフォーミュラ確認

例: I16 は `=SUM(I4:I15)`。新月行を埋めると合計が自動更新されるので、想定通りの値か確認するだけでOK。

## 来月以降の運用

来月末（5月→6月）に「先月と同じ月次まとめお願い」と Claude に伝えれば、本手順書通りに実行する。月数字を読み替えるだけ。

スナップショットを使った検証フロー:
1. `node snapshot.mjs before` で作業前を保存
2. ユーザーが Step 1〜6 を実行
3. `node snapshot.mjs after` で作業後を保存
4. `node diff.mjs before after` で想定通りに変更されたか確認
